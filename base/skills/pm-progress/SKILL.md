# PM Progress Tracking

## Источник знаний
- `docs/server_docs/development_progress.md` — журнал прогресса
- `docs/server_docs/core_structure_execution_checklist.md` — чеклист выполнения
- `docs/server_docs/migration_contract.md` — миграционный контракт

## Когда применять
- обновление прогресса разработки
- создание чеклистов для новых фич
- аудит статуса по этапам
- подготовка отчётов о завершённых/блокирующих задачах
- road-map планирование

## Процесс трекинга

### 1. Журнал прогресса
Файл: `docs/server_docs/development_progress.md`
- Добавлять записи по завершении крупных блоков
- Формат: дата + что сделано + статус
- Не фиксировать каждую мелкую правку — только значимые вехи

### 2. Чеклисты
- `core_structure_execution_checklist.md` — для проверки core-структуры
- Для новых фич создавать отдельные чеклисты в `docs/tmp/`
- Каждый пункт: задача → ответственный → статус → примечание

### 3. Планирование
- P0: блокирующие дефекты (дни)
- P1: существенные проблемы (неделя)
- P2: улучшения (месяц)
- Оценка трудозатрат в днях/человеко-днях

### 4. Отчёты
- По запросу: сводка по приоритетам, рискам, блокерам
- Хранить в `docs/tmp/` как временные файлы

## Инварианты
- Не хранить временные планирования в корне — только `docs/tmp/`
- `development_progress.md` — только крупные вехи
- Чеклисты должны быть actionable (конкретные шаги, не абстракции)
